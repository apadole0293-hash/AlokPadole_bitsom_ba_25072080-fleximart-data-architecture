# Part 2: NoSQL Database Design (MongoDB)

This section presents the NoSQL data modeling approach for the FlexiMart retail system using MongoDB.

The objective of this part is to demonstrate how semi-structured and hierarchical data can be modeled using a document-oriented database. The design focuses on flexibility, scalability, and efficient read operations.

## Contents
- `nosql_analysis.md` – Detailed MongoDB schema design and justification
- `products_catalog.json` – Sample product catalog documents
- `mongodb_operations.js` – Example MongoDB queries and operations
